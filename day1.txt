Основы работы с системой контроля версий Git
Цель: Сформировать базовые практические навыки работы с Git через Git Bash в VS Code: создание репозитория, отслеживание изменений, создание коммитов и анализ истории проекта.

1) Установка и настройка, сформировать понимание ключевых понятий, обучить основным командам.
2) Аккуратность при работе в терминале, культуру инженерного труда и ответственность за сохранность работы.
3) Навыки работы с терминалом, анализа истории изменений, системное мышление.

Система контроля версий (VCS) — программное обеспечение, отслеживающее изменения в файлах проекта с течением времени. Позволяет вернуться к любой версии, сравнивать изменения, работать в команде без риска потери данных.
Git был создан в 2005 году Линусом Торвальдсом — автором ядра Linux. Причина: необходимость в быстрой и распределённой системе контроля версий. Сегодня Git — стандарт индустрии: его используют Google, Microsoft, Meta, Yandex.
Modified(изменён) - Файл изменён в рабочей директории, но не добавлен в индекс
Staged (проиндексирован) - Файл добавлен командой git add и готов к коммиту
Committed (зафиксирован) - Изменения сохранены в репозитории командой git commit
